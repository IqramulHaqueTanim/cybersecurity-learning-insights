# TryHackMe: Pre-Security Path Overview

**Path Completed:** May 15, 2026<br>
**Documented:** May 27, 2026

## Overview
Since I already have my CSE degree, a lot of this path served as a solid refresher. I decided not to document every single guided task since the modules were heavily hand-held and meant for complete beginners. Instead, I used this path to connect the dots between the general computer science concepts I already know and how they actually apply to defensive and offensive cybersecurity. 

Here is a quick summary of my main takeaways and the baseline concepts I validated before tackling more advanced material.

## Core Insights

### 1. Networking & The Web
* **The Security Angle:** I already knew the OSI model, TCP/IP, and the three-way handshake, but this time I focused on how data encapsulation matters when you are trying to spot network vulnerabilities.
* **Network Infrastructure:** Brushed up on LAN topologies, subnetting, ARP, and DHCP. I also reviewed how port forwarding and VPNs impact a network's overall attack surface.
* **Web Mechanics:** Looked at how HTTP headers, status codes, and DNS domain hierarchy tie directly into common web weaknesses, like HTML injection and sensitive data exposure.

### 2. Operating Systems
* **System Privileges:** Reviewed how operating system privilege layers dictate the security of the entire system, focusing on how native security controls operate under the hood.
* **Command Line:** Did a practical review of navigating both Linux and Windows environments through the CLI, focusing on gathering system info and managing files without relying on the GUI.
* **Virtualization & Cloud:** Covered the basics of deploying cloud instances and managing VMs, which mirrors how most modern enterprise environments are set up.

### 3. Data, Scripting & Cryptography
* **Encoding:** Reviewed how data is represented under the hood (Hex, ASCII, Unicode). Being comfortable with this will be important later for analysing raw network traffic and payloads.
* **Scripting Context:** Did a quick run-through of Python, JS, and SQL basics. The goal wasn't to learn how to code, but to make sure I can read and understand application logic and database tables during security assessments.
* **Security Mindset:** Finished up by anchoring everything to the CIA Triad and making sure I fully understood the mechanics of symmetric vs. asymmetric encryption.
